連線到執行個體的方式

Posted on 2016-04-12 Edited on 2023-11-24 In SQL SERVER Views: Disqus:

 

一、以一開始安裝選項為具名執行個體、使用動態通訊埠49674，名稱為SQLEXPRESS為例

如果要連到本機伺服器並指定執行個體名稱為SQLEXPRESS可以使用以下方式

1、「.」代表本機伺服器。

2、「Localhost」代表本機伺服器，而且不分大小寫。

3、「(local)」代表本機伺服器，而且不分大小寫，簡寫local外的括號不可拿掉。

4、「127.0.0.1」代表本機伺服器。

5、指定本機伺服器名稱「DESKTOP-RF9FL3I」代表本機伺服器，而且本機伺服器名稱不分大小寫。

6、使用對外IP「219.84.252.110」代表本機伺服器。

7、使用對內IP「169.254.230.223」代表本機伺服器。

8、本機伺服器加上埠號加上指定執行個體名稱為SQLEXPRESS也可連到本機，而且執行個體名稱不分大小寫。

 

註：如果指定埠號而不指定執行個體名稱SQLEXPRESS也可連線

 

註：如果同時不指定埠號與執行個體名稱SQLEXPRESS將無法連線

 

註：每次啟動SQL SERVER才替具名執行個體指定通訊埠號，

所以通訊埠號每次啟動都是不同埠號，此稱之為動態通訊埠。

 

二、以一開始安裝選項為預設執行個體、使用靜態通訊埠1433，名稱為MSSQLSERVER為例

如果要連到遠端伺服器的MSSQLSERVER執行個體可以使用以下方式

1、使用對外IP「219.85.115.243」代表遠端伺服器，預設連到MSSQLSERVER執行個體。

2、使用對外IP加上埠號，預設連到MSSQLSERVER執行個體。

3、使用對外IP加上埠號加上執行個體名稱MSSQLSERVER，預設連到MSSQLSERVER執行個體。

4、使用對外IP加上埠號加上執行個體名稱SQLEXPRESS，預設連到MSSQLSERVER執行個體。

註：對外IP不加埠號只加上執行個體名稱，將無法連線
